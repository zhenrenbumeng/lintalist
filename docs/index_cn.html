简介


如前所述，Lintalist 允许你将文本存入 Bundle，并对它们进行（增量）搜索与编辑，再将所选文本粘贴到当前活动程序中。

这些文本、快捷键、缩写与脚本统称为 Snippet，它们保存在所谓的 Bundle 中。Bundle 是简单的文本文件，因此易于理解、维护，也便于在用户之间交换。

Bundle 具有上下文感知能力：根据当前活动窗口的（部分）标题，程序会尝试加载一个或多个匹配的 Bundle，从而让你在不同应用程序与文件中使用相同的快捷键与 Shorthand，更加灵活。你不必记住或使用任何快捷键或缩写，因为随时可以通过打开搜索窗口来搜索 Bundle。把它当作一款文本扩展器：能省下大量按键，并提供诸多附加功能：Hotkey、全文搜索、Plugin 与脚本。
有人也会把它比作许多文本编辑器里的「标签列表」或「剪贴库」。

Bundle 与 Snippet 可使用 Lintalist Bundle 与 Snippet 编辑器编辑，也可以在你偏好的文本编辑器中编辑。

例如：若正在编辑 HTML 文件，Lintalist 可同时使用 HTML 与 CSS 的 Bundle；若正在编辑 CSS 文件，则只加载 CSS Bundle。因此在 HTML 文件中，快捷键 Ctrl+B 可在光标处插入 <strong></strong>（或将标签套在选中文本外）；而在编辑 CSS 文件时，快捷键 Ctrl+B 可在光标处插入 font-style: bold;。若在多个活动 Bundle 中找到同一快捷键，会弹出菜单供你选择要使用的 Snippet 与 Bundle。

注意：虽然 Lintalist 可以展开缩写，但并不旨在复制或取代 AutoHotkey 原生的 Hotstrings 功能；选项说明请参阅 autohotkey.com/docs/v1/Hotstrings.htm。

若觉得不便，可编辑某个 Snippet 并更改其快捷键或 Shorthand。
本程序使用 AutoHotkey 编写；AutoHotkey 是适用于 Windows 的免费、开源（脚本）工具。更多信息见 autohotkey.com。

Lintalist 是一款便携应用。它将所有设置与 Bundle 存储在程序所在文件夹中。

首次运行时会询问是否在桌面创建快捷方式或随 Windows 启动，但你可以不做这些设置。也可以从 U 盘或 Dropbox 等位置直接运行。

若要随身携带 Lintalist，只需复制整个 Lintalist 文件夹；备份同理。

安装 Lintalist


安装 Lintalist 很简单：下载 ZIP 并解压到你选择的文件夹。请确保目标文件夹可写（例如不要解压到 “C:\Program Files” 这类目录）。放在「我的文档」下通常最省事，这样在使用时 Lintalist 也更容易更新你的 Bundle。Lintalist 完全便携，不会向注册表写入设置，因此可直接从 U 盘或 Dropbox 文件夹等位置运行。

若你已安装 AutoHotkey，建议运行 Lintalist.ahk，这样会使用你本机偏好的 AutoHotkey 版本运行 Lintalist（推荐 v1.1.37.02）。若未安装 AutoHotkey，可直接启动 Lintalist.exe。

说明：
- Lintalist.exe：重命名后的 AutoHotkey.exe（v1.1.37.02 32/64 位 Unicode，见 https://autohotkey.com/ 与/或 https://autohotkey.com/download/）
- Lintalist.ahk：程序的实际（主）源代码（见 https://github.com/lintalist/lintalist）

Scoop：https://scoop.sh/#/apps?q=lintalist（未测试，不提供支持）


常规使用：启动 Lintalist 并搜索 Snippet


若 Lintalist 正在运行，可按 CAPSLOCK（默认 Hotkey）激活搜索窗口。可在托盘图标  上右键打开配置菜单以更改该 Hotkey。

根据活动程序标题，会有一个或多个 Bundle 加载到搜索窗口，随后即可开始搜索：

Lintalist 搜索窗口：

1：输入查询。
2：菜单与搜索选项。
3：结果列表，含 Part1、Part2、Key（键盘快捷键）、Short（缩写）。列为动态：若已加载的 Bundle 中不存在 Part2、Key 或 Short，则不显示对应列。
4：最上方或所选 Snippet 文本的预览。状态栏显示：已加载的 Bundle，以及命中数 / Snippet 总数。

若在 Lintalist 搜索窗口已激活时再次按同一 Hotkey，会在宽视图与窄视图之间切换。可在配置菜单中分别设置两种视图的宽度与高度。

输入时结果数量会减少（即增量搜索或快速过滤）。小图标表示纯文本 Snippet ，或会实际运行脚本的 Snippet 。可在配置菜单中开关图标（ShowIcons），也可通过排序选项影响结果顺序（见 SortByUsage 与 ColumnSort）。

在结果列表中选中一项后按 ENTER，默认将文本（Part1）粘贴到活动程序（见上文说明）。若未用方向键选择其他项，SHIFT+ENTER 与 ENTER 均作用于列表中的第一项。根据 Snippet 不同，会粘贴文本或运行脚本（见上文说明）。
若只想将 Snippet 复制到剪贴板而不粘贴：按 Alt+ENTER 复制 Part1，按 Alt+Shift+ENTER 复制 Part2。

也可用 Up 与 Down 在列表中移动。可通过 Bundle 菜单选择其他 Bundle 或加载所有 Bundle。若从 Bundle 菜单（Alt+b）选择某个 Bundle，该 Bundle 将被锁定（工具栏图标：），并一直保持激活，直到你解锁（Alt+l，字母 l）。
若启用该功能，可使用 Alt+1..0 快速选择前十个搜索结果。

（双击）点击列表项时的行为由配置菜单中的设置决定。（SingleClickSends 与 DoubleClickSends）

默认 Bundle：另有始终加载的默认 Bundle——除非你锁定了特定 Bundle——这样在每个程序中都有通用 Snippet 可用。

若不需要上下文感知的 Bundle，只需使用默认 Bundle 并将所有 Snippet 放入其中。此时也可锁定默认 Bundle，但通常无此必要。

搜索 Snippet 的哪些部分？


程序会在 Part1、Part2 与 Shorthand 中查找匹配（不会搜索快捷键与脚本代码）。为在结果列表中显示更易读文本，若 Part1 与 Part2 为多行，其展示方式会有所不同：

换行替换为 \n，制表符替换为 \t，搜索针对这些文本而非原始多行内容（粘贴时仍使用原始文本；多看几次实际操作即可习惯）。


RegEx 与大小写敏感搜索


<技术细节> 若大小写复选框未勾选，搜索不区分大小写。使用 RegEx 搜索时，会在搜索每个 Snippet 之前自动为正则插入 "i)" 选项。若你为 RegEx 搜索传入其他选项，可能导致意外结果。

示例：不区分大小写的 RegEx 'se\w*g' 在搜索时实际使用 'i)se\w*g'。这样更省事（否则每次不区分大小写都要输入 i)，而 RegEx 默认区分大小写）。若要传入额外选项，建议开启大小写敏感，再按此处说明添加所需选项：autohotkey.com/docs/v1/commands/RegExMatch.htm#Options。此行为在 Lintalist 未来版本中可能变更。</技术细节>

搜索字母变体

有时搜索文本中含有带重音的字母，输入可能困难或较慢。将这些字母的变体加入搜索串后，可搜索 'cafe' 仍匹配 'café'。因此查询中的 'e' 会扩展为搜索 'eéèêḙěĕẽḛẻėëēȩęᶒɇȅếềễểḝḗḕȇẹệæǽǣœᵫ'。
启用后，所有搜索模式都会使用正则表达式（常规与模糊）。
可通过搜索界面中的 Lv 按钮（Alt+v）或配置菜单中的 SearchLetterVariations（取消）启用。
更多说明见 'include\LetterVariations.ahk' 中的注释。


OmniSearch（搜索所有 Bundle）


默认只在已加载的 Bundle 中搜索，也可用以下方式在所有 Bundle 中搜索：

1. 加载所有 Bundle

可加载全部 Bundle：

- 使用托盘菜单，选择「加载所有 Bundle」，再打开搜索界面；或
- 打开搜索界面，Bundle 菜单第一项：加载所有 Bundle

缺点是也会设置锁定状态，因此下次打开搜索界面时仍会在所有 Bundle 中搜索，需手动解锁。

2. OmniSearch 查询

当查询以预定义字符开头（默认为 @）时，Lintalist 会在所有 Bundle 中搜索，不受当前已加载或锁定的 Bundle 限制。可在设置中定义该字符（OmniChar）。若不想在输入查询时使用 OmniSearch，在设置菜单中清空 OmniChar 即可禁用。

查询：state -> 在当前已加载的 Bundle 中查找 "state"
查询：@state -> 在所有 Bundle 中查找 "state"

3. OmniSearch 快捷键

- 搜索界面激活时：按 F2 切换 OmniSearch。
- 按 Ctrl+Capslock 时，搜索界面以 OmniSearch 模式打开（提示：用 F2 切换）。可在设置中定义该 Hotkey：StartOmniSearchHotkey

注意：使用选项 2 与 3 时，结果更新后状态栏（右下角）会附加文本 "(All)"，表示正在使用 OmniSearch。


ColumnSearch（搜索 Snippet 的特定部分）


默认情况下，Lintalist 会在 Snippet 的 Part1、Part2 与 Shorthand 中搜索匹配。

当查询以数字（1 到 6）开头，后接预定义字符（默认为 <，设置中为 ColumnSearchDelimiter）时，Lintalist 只在 Snippet 的特定部分中搜索。要禁用此功能，请在设置中清空 ColumnSearchDelimiter。

查询：state -> 在 Snippet 的 Part1、Part2 与 Shorthand 中查找 "state"
查询：1<state -> 仅在 Snippet 的 Part1 中查找 "state"

1 到 5 对应 Snippet 的「部分」或「区段」（6 对应 Bundle 名称与描述，若有）：

1< Part1
2< Part2
3< Shortcut（快捷键）
4< Shorthand（缩写）
5< Script（脚本）
6< Bundle 名称与描述

在查询开头加上 OmniSearch 字符可与 ColumnSearch 组合（例如 @1<state）


锁定 Bundle 与 TitleMatch


默认情况下，Bundle 使用 TitleMatch 属性决定使用哪些 Snippet。要始终使用特定 Bundle：

1. 始终使用所有 Bundle——见加载所有 Bundle
2. 从 Bundle 菜单加载特定 Bundle。
3. 使用锁定工具栏图标：，或快捷键 Alt+l

另见 FAQ 5。


使用快捷键（Hotkey）


若 Snippet 定义中包含快捷键，且该 Snippet 属于当前活动的 Bundle，可在不激活 Lintalist 搜索窗口的情况下，通过按下 Hotkey 插入 Part1 的文本或运行脚本。Lintalist 搜索窗口激活时，快捷键被禁用。

可通过托盘菜单或搜索窗口工具栏中的按钮启用/禁用快捷键


使用缩写（Abbreviations）


若 Snippet 定义中包含 Shorthand，且该 Snippet 属于当前活动的 Bundle，可在不激活 Lintalist 搜索窗口的情况下，通过输入缩写后按 Space 或 Tab 插入 Part1 的文本或运行脚本。Lintalist 搜索窗口激活时，Shorthand 被禁用。

可通过托盘菜单或搜索窗口工具栏中的按钮启用/禁用 Shorthand

Lintalist Bundle


Lintalist Bundle 的基本结构

虽然很多支持 Bundle、Taglist、Clip-list 或 Snippet 的程序喜欢使用 XML 或 INI，Lintalist Bundle Format 只是格式化的普通文本文件，易于理解和编辑。它的结构类似 YAML，不像 XML 那样笨重。你可以使用 Lintalist 内置编辑器 编辑、复制、移动、添加、删除 Snippet，也可通过 Bundle 属性编辑器 从零创建新 Bundle。还可以使用 Bundle 转换器 从其他应用导入数据。

Bundle 文本文件保存在 bundles\ 目录。开始编辑 Bundle 后，备份会复制到 bundles\backup\。

代码与参数说明




^| 或 ^$粘贴后的光标位置。例如 “Hello, this is a sample ^|text” 会移除 ^|，并把光标放到 text 前。^$ 是替代写法，适合与 Choice 插件组合，因为 | 字符会冲突。若目标编辑器支持 Multi-caret/edit，可在 Snippet 中放多个光标位置。Unicode、复杂文字、自动缩进或目标应用格式化代码时，光标位置可能不准确。


[[C=Char|Repeat]]默认粘贴文本会 trim 首尾空格和 Tab。此插件可在 Snippet 开头或结尾插入空格、Tab、换行，也可重复字符或文本。例如 [[C=-|30]] 插入 30 个连字符。可用 [[C=space]]/[[C=\s]] 表示空格，[[C=tab]]/[[C=\t]] 表示 Tab，[[C=enter]]/[[C=\n]] 表示换行。


[[Calc=]] *2计算数学表达式，详见下方 Calc 文档。


[[Calendar=Format|LCID|to]]显示日历窗口，让你选择日期并按指定格式插入。格式见 DateTime。示例：[[Calendar=dd-MMM-yyyy]] 会插入类似 25-Jan-2011。可指定 LCID；若选择日期范围，默认用 “ to ” 连接，也可用第三参数指定其他连接文本，如 [[Calendar=dd-MMM-yyyy|| - ]]。


[[Choice=item1|item2|..]]从列表选择项目，选项用 | 分隔。例如 [[Choice=Red|Green|Blue]]。若第一个选项以 ? 开头，它会作为提示文本显示；若以 ! 开头，可边输入边过滤选项；?! 顺序无关。可启用 “Select by digit” 用 Alt+1..0 选择前 10 项。


[[Clipboard]] *1插入当前剪贴板内容。额外参数见 扩展剪贴板与选中文本。


[[Comment=]]给 Snippet 添加注释。注释可被搜索，但粘贴前会完全移除。也可使用 Part2 作为提示或备注。


[[Counter=Name|Value|Reset]]跨 Bundle 计数器。示例 [[Counter=InvoiceNumber]] 默认加 1 并插入值；[[Counter=InvoiceNumber|5]] 加 5。若同一 Snippet 多次使用同一计数器但不想递增，后续实例可用 |0。计数器保存于 settings.ini，可通过托盘菜单 Manage counters 管理。未知计数器会自动创建。可用第三参数 reset 重置初值。


[[DateTime=Format|Value|TimeUnits|LCID|RollOver]]按 AutoHotkey FormatTime 选项插入当前日期/时间。示例 [[DateTime=dd-MMM-yyyy]]。可计算时间：[[DateTime=dd-MMM-yyyy|5|Days]] 插入 5 天后的日期。TimeUnits 可为 Seconds、Minutes、Hours、Days。LCID 可控制区域语言。RollOver 可把日期滚动到指定星期，如 mon-fri 或 2-6。


[[File=Filename|select|clean]]把指定文件内容插入文本。路径可为完整路径或相对于 Lintalist 目录。|Select 会打开文件选择对话框，Filename 作为初始目录；|Clean 会移除 []| 字符，适合把文件内容作为另一个插件输入时使用。Select 与 Clean 顺序不限。


[[FileList=?Options|path]]从一个或多个文件夹返回分隔文件列表，常与 [[Choice]] 配合选择文件。支持 * 和 ? 通配符，多个通配符用 ; 分隔。选项：D 包含目录，F 包含文件，R 递归，P 完整路径，SC 逗号分隔，SN 换行，ST Tab，T Total Commander 当前面板，E Explorer 当前文件夹，W 当前窗口标题路径，U URI 路径使用 /，X 返回相对路径，! 自动用于可过滤 Choice。示例：[[FileList=!?P|[[A_ScriptDir]]\docs\img\*.*]]，可用 F3 切换预览。


[[Input=Text|Default]]请求用户输入。示例：[[Input=Customer's name?]]。同一问题重复出现时会复用输入。可用第二参数提供默认值，如 [[Input=My favourite color|Green]]。


[[PasteMethod=0|1|2]]类似全局 PasteMethod，但只作用于当前 Snippet。0 粘贴并保留剪贴板，1 粘贴并把 Snippet 留在剪贴板，2 不粘贴只复制到剪贴板。

[[Selected]] *1替换为当前选中文本。可用于把选中文本包裹进 HTML 标签等。额外参数见 扩展剪贴板与选中文本。


[[Snippet=Shorthand|?part]]在指定位置插入同一 Bundle 中另一个 Snippet 的内容。使用目标 Snippet 的 Shorthand。默认插入 part1；|2 插入 part2，|3 表示脚本，|4 插入 shorthand。


[[Split=input|?1|?2]] *3把输入拆分为变量，详见下方 Split 与 SplitRepeat。


[[SplitRepeat=input|?1|?2|?3]] *3拆分并重复处理输入，详见下方 Split 与 SplitRepeat。


[[Var=Name]] *4插入跨 Bundle 可用的本地变量。详见 本地 Bundle 变量。本地变量也可在脚本中使用。


[[String=text|option]]转换文本。见 扩展剪贴板与选中文本。String 插件不支持 wrap 选项。例如 [[String=[[Input=Name?]]|Upper]] 会把输入姓名转为大写。


[[Part1]]、[[Part2]]在另一个部分中插入当前 Snippet 的 Part1 或 Part2。例如 [[choice=[[part2]]]] 可把 Part2 作为 Choice 选项。为避免递归，part1 会从 part1 中移除，part2 会从 part2 中移除。


[[Query]]、[[Query1]]、[[Query2]]把你在搜索框输入的查询文本插入 Snippet。Query 插件先于其他插件处理。默认 QueryDelimiter 是 >，搜索时该字符之后内容会被忽略；Query1 是分隔符之前，Query2 是之后。通过快捷键或缩写触发且没有搜索窗口时，Query 会自动转换为 Input 请求输入。


[[html]]、[[md]]、[[rtf=]]、[[image=]]用于 格式化文本 Snippet。提示：Image 插件接受 clipboard 作为图片路径，例如 [[image=[[clipboard]]]]。可配合 [[PasteMethod=2]] 让图片保留在剪贴板中。


[[llpart1]]、[[llpart2]]、[[llshorthand]]在 脚本中作为 Snippet Part1、Part2、Shorthand 的占位符。

[[Clipboard=options]] 与 [[Selected=options]]




选项缩写用途

*2 Calc 插件


Calc 插件使用 Laszlo 的 Monster/Eval 函数计算数学表达式。它支持基础与高级数学运算、标准 AutoHotkey 函数、预定义函数和常量。

- 算术运算符：+、-、*、/、\（或 % = mod）、**（或 @ = power）
- AHK 函数：Abs、Ceil、Exp、Floor、Log、Ln、Round、Sqrt、Sin、Cos、Tan、ASin、ACos、ATan
- 预定义函数：SGN、Fib、Fac（sign、Fibonacci、Factorial）
- 预定义常量：e、pi、inch、foot、mile、ounce、pint、gallon、oz、lb




[[calc=]] 中的表达式结果


1 + 12


6 / 23


12 + 3 * 4 / 2 - 117


5 \ 21（5 mod 2 = 1）


100/3 $233.33（保留 2 位小数）




输入示例：



Converting Celsius to Fahrenheit °C x 9/5 + 32. Today it was:
[[Calc=([[Input=What is todays temperature in C?]]*(9/5))+32]]°F (=[[Input=What is todays temperature in C?]]°C)



使用输入与本地变量：

*3 Split 与 SplitRepeat


[[Split=input|?1|?2]] 与 [[SplitRepeat=input|?1|?2|?3]]

Split 会创建一组按顺序编号的临时变量，可放在 Snippet 任意位置。这适合处理从表格复制来的多列或多行文本。




选项含义


input可以是文本、另一个插件，或保留关键字 selected / clipboard。注意 input 不应包含 []|。


?1第一层拆分字符。保留选项：\s = 空格，\t = Tab，\n = 换行。只使用 ?1 时，会创建 [[sp=1]]、[[sp=2]]、[[sp=3]] 等变量。


?2第二层拆分字符，创建二维变量，如 [[sp=1,1]]、[[sp=1,2]]、[[sp=3,2]]。使用 ?1 与 ?2 时，?1 可理解为行标记，?2 为列标记。




示例，选中文本为 word1 word2 word3：



[[Split=selected|\s]]
Hello [[sp=1]]
Hello [[sp=2]]
Hello [[sp=3]]



结果：



Hello word1
Hello word2
Hello word3



读取文本文件并使用第三行：



[[Split=[[File=example.txt]]|\n]]
The third line of the text file is [[sp=3]]



二维示例，选中文本：

wordA1 wordA2 wordA3
wordB1 wordB2 wordB3

使用：



[[Split=selected|\n|\s]]
Hello [[sp=1,1]]
Hello [[sp=1,2]]
Hello [[sp=1,3]]
Hello [[sp=2,1]]
Hello [[sp=2,2]]
Hello [[sp=2,3]]



SplitRepeat 类似 Split，但能重复 Snippet，直到所有部分都被处理完。它最多接受三个拆分参数 ?1|?2|?3。适合输入行数或项目数未知的场景。

基本示例，选中文本为 a,b,c,d：



[[SplitRepeat=selected|,]]
Hello [[sp=1]][[C=\n]]



结果：



Hello a
Hello b
Hello c
Hello d



命名 Split

可写为 [[Split_name=input|?1|?2]] 与 [[SplitRepeat_name=input|?1|?2|?3]]。如果一个 Snippet 中需要多个 Split，命名可避免互相干扰，也允许 Split 与 SplitRepeat 组合。

例如：



[[Split_MyName=selected|\t]]
[[SplitRepeat_OtherName=[[sp_MyName=2]]|\s]]
result=[[sp_OtherName=1]]



还可以把 Split 与 Choice 组合，实现带逻辑的 Snippet：先用 Choice 选择编号，再用编号从另一个 Split 中取对应值。

函数


此功能面向更有经验的 AutoHotkey 用户。Lintalist 支持 AutoHotkey 内置函数（Asc、SubStr、Floor、Mod 等）以及用户自定义函数。

调用语法：[[function(optional parameters)]]。不要写成 function=()，函数名后不要加等号。

示例：[[SubStr("AutoHotkey",5,3)]] → 结果：Hot

自定义函数需要被包含进 Lintalist。可以写在 plugins\MyFunctions.ahk 中，也可以在该文件中 #include 其他函数。更新 Lintalist 时 MyFunctions.ahk 不会被覆盖。修改后请重启 Lintalist。

标准函数示例：[[EnvGet(EnvVarName)]]，可读取 Windows 环境变量。

如果函数需要更新整个 Snippet，而不只是处理返回值，需要在函数中把 clip 声明为 global：



MyFunc(parameters)
   {
    global clip
    ; ... your code ...
    clip:="New Content"
    Return
   }

格式化文本 Snippet


Lintalist 支持多种格式化文本 Snippet。格式化文本或富文本不同于纯文本，包含颜色、粗体、斜体、字号、链接等样式信息。

通用注意事项：
1. 粘贴目标应用必须支持对应格式，否则不会粘贴任何内容。例如把图片粘贴到 Notepad 不会有结果。某些应用接受 HTML 但不接受 RTF，反之亦然。
2. HTML、Markdown 与 RTF 支持插件，Image 不支持。RTF 中的插件支持取决于 RTF 文件写法（换行、格式等），可能需要实验，并尽量避免给插件文本本身套样式。
3. 粘贴后光标位置 ^| 不适用于这些格式化文本 Snippet。
4. 如果显示图标，列表中会用 HTML/Markdown、RTF、Image 图标标识格式化 Snippet。


HTML


要把 HTML 作为格式化文本粘贴，需要在 Snippet 中包含 [[html]]，通常放在开头，但可位于任意位置。可以省略 html、head、body 标签。

可用 [[Snippet=]] 嵌套或链接多个 HTML Snippet。如果任何嵌套 Snippet 包含 [[html]]，整个 Snippet 都会按 HTML 转换。不要混用 HTML 与 MD Snippet，以免结果不可预期。



[[html]]
<h3>Hello: [[Input=What is your name]]</h3>

<p>This <b>is</b> a <em>sentence</em> with formatting.</p>

<p>Today is <em>[[DateTime=dd MMMM yyyy]]</em>.</p>

<ul>
	<li>list item</li>
	<li>list item</li>
	<li>list item</li>
</ul>

<p>Images also work - either use an http URI:<br />
<img src='http://lintalist.github.io/img/lintalist.png'></p>

<p>or a file URI:<br />
<img src='docs/img/lintalist.png'></p>



注意：某些应用（例如 Webmail）需要 HTTP URI 才能使用图片。若要使用本机图片，可以省略完整 file:// 路径，Lintalist 会基于安装目录自动补全相对路径。

示例：

	<img src='Images/lintalist.jpg'>
	<img src='../../anotherimage.png'>


Markdown


要把 Markdown 作为格式化文本粘贴，需要在 Snippet 中包含 [[md]]。目前只支持基础 Markdown；嵌套列表尚不支持，表格支持也较基础。可以嵌套或链接多个 MD Snippet，但不要与 HTML Snippet 混用。



[[md]]
# Heading 1

* unordered list entry
* unordered list entry

this is a **normal line** of *text* with some markup

1. numbered list entry
2. numbered list entry

Inserting a link [AutoHotkey](https://autohotkey.com) also works

![Lintalist](https://raw.github.com/lintalist/lintalist/master/docs/img/lintalist.png)



Markdown 转 HTML 函数来自 GenDocs 的简化 Markdown 语法，支持：段落、行尾两个空格的换行、最多三级标题、强调、加粗、行内代码、以 > 开头的代码区块、无序列表、有序列表、转义序列、链接、图片，以及基础表格。

插入 HTML 块可用两段 ~~~ 包围：

~~~
<p>HTML code here</p>
~~~

表格规则：以 | 开头表示表格开始，空行表示结束；第一行单元格视为表头；单个 | 表示单元格分隔；不支持单元格对齐、跨列、跨行。

RTF 类似 File 插件，会从指定文件插入格式化文本（包括图片）。路径可以是完整路径或相对于 Lintalist 目录。Snippet 中其他文本会被丢弃。


[[rtf=Extras\examples\example.rtf]]


示例 RTF 文件可在 Extras\formatted\ 或 extras\examples\ 中找到。

新建 Bundle

有两种添加新 Bundle 的方式：

1. 在文本编辑器中创建新 Bundle，并复制到 \bundle\ 文件夹后重新加载。
2. 使用 Lintalist 创建 Bundle 并开始添加 Snippet。

打开搜索窗口，使用 Edit → Bundle properties 或按 F10。Lintalist Bundle Editor 打开后可输入名称等信息。点击保存时会要求输入 Bundle 文件名。

最重要字段是 Name 与 TitleMatch。Name 用于 Bundle 菜单，也可显示在搜索结果中；TitleMatch 可让 Bundle 上下文敏感，也就是只有窗口标题匹配时，搜索、缩写和快捷键才可用。

TitleMatch 可以是逗号分隔列表：

1. 窗口标题的部分匹配。不要使用通配符。示例：.txt,.doc 可以，*.txt,*.doc 不可以。
2. 程序可执行文件名，即 AutoHotkey 中的 ahk_exe。示例：cmd.exe,notepad.exe。

两者可组合，例如：.txt,notepad.exe。

反向匹配：TitleMatch 以 ! 开头时，表示这些窗口中 Bundle 不应激活。例如 !txt 让 Bundle 在除 txt 文件外的窗口中可用。此功能测试程度不如常规 TitleMatch。

新 Bundle 保存在 bundle 目录中。添加新 Bundle 后 Lintalist 会重启，并自动加入第一个 Snippet。之后可通过 Edit → New snippet 或 F7 添加更多 Snippet；请先通过搜索窗口 Bundle 菜单加载目标 Bundle。完成后别忘了解锁。

-Ini=filename-for-settings.ini：使用不同设置文件（默认 settings.ini）。

-ReadOnly：只读模式启动，不允许编辑 Bundle、settings.ini、计数器和本地变量。Lintalist 搜索 GUI 标题与托盘提示中会添加 *。

-Administrator：以管理员身份启动/重启 Lintalist。也可通过 Configuration 中的 Administrator 设置或托盘菜单使用。Lintalist 搜索 GUI 标题与托盘提示中会添加 username^。


F.A.Q 与已知问题



FAQ


Q1：没有粘贴任何内容 / 只看到字母 v（或什么都没有）
A1：你可能需要增加 PasteDelay。右键托盘图标，Configuration，PasteDelay，增加毫秒数。可能需要多试几次找到合适值。

Q2：仍然不工作！
A2：可尝试不同 SendMethods。共有 4 种：右键托盘图标，Configuration，SendMethod。逐个尝试，必要时与 PasteDelay 组合测试。Notepad 是很好的测试程序。

Q3：我的编辑器在没有选中文本时也会选择整行。
A3：有些编辑器在使用 [[selected]] 或脚本时，即使未选中任何内容，也会自动选择整行（甚至空行）。已知编辑器包括 UltraEdit（可在配置中关闭）、EditPad、Sublime Text 等。如果不能关闭该行为，请先选中文本再使用 Snippet，或从 Snippet 中移除 [[selected]]。

Q4：我想做 OR 搜索，例如 “this OR that”，可以吗？
A4：可以。选择 RgEx（正则表达式）搜索（Alt-R）并使用 | 管道符。例如 this|that 会找到文本（part1、part2、shorthand）中包含 THIS 或 THAT 的 Snippet。

Q5：输入 shorthand 或按 shortcut 后没有反应！
A5：
- 确认 shortcuts 与 shorthand 未被禁用，Lintalist 也没有暂停。
- 可能锁定了另一个 Bundle，导致你认为应激活的 Bundle 并未激活；请用工具栏按钮解锁。
- 如果不是默认 Bundle，请确认该 Bundle 的 TitleMatch 设置正确。


已知问题


K1：无法向 Bundle 追加新 Snippet。
A1：尝试 Reload Bundles 或重启 Lintalist。某些情况下 Lintalist 可能找不到或无法访问 Bundle，或丢失了应追加到哪个 Bundle 的状态。v1.9.5+ 后理论上不应再发生；若你遇到，尤其能复现，请报告。

K2：Flash 编辑控件不总是可用。
A2：Lintalist GUI 激活时 Flash 编辑控件会失去焦点。无法可靠检测并重新聚焦这类控件。如果经常遇到，可把 PasteMethod 改为 1 或 2 后手动粘贴。

K3：不能与 Windows Store 安装的程序配合。
A3：这是已知情况，目前没有解决方案。你也可以尝试手动粘贴，见 PasteMethod 设置，或使用 PasteMethod 热键。

RunQuery

允许用户使用搜索框中的查询内容，启动用户自定义脚本，并把查询作为参数传递。

settings.ini 中有三个隐藏选项：

QueryAction：禁用/启用
QueryHotkey：用于“run query”的热键
QueryScript：要启动的脚本，查询会作为参数传给它。默认 RunQuery.ahk；如果启用 QueryAction 且未找到脚本，会创建一个占位脚本。
